# AmekoRyTheater Roadmap

本文件用于记录 AmekoRyTheater 后续开发路线、功能想法、优先级和取舍判断。

项目当前定位：

* 个人创作者内容站
* 免费作品展示
* 单篇付费作品
* 用户注册登录
* 购买后解锁隐藏内容
* 评论互动
* 管理员后台发布和管理内容

当前不做包月会员制，不区分当月作品 / 过往作品，优先以“单篇付费解锁”为核心模式。

## 近期最高优先级

这些内容会直接影响网站观感、正式感和基础体验，应优先处理。

### 1. 全站文案统一

当前问题：

* “买断”一词不准确，容易被理解为买下作品版权。
* 应改成“付费”“购买后解锁”“已购买”“付费内容”等表达。

处理方向：

* 将“买断 ¥xx”改为“付费 ¥xx”
* 将“买断作品”改为“付费作品”
* 将“永久解锁”保留，但避免让用户误解为拥有版权
* 在购买说明中明确：购买后仅获得个人查看权限，不代表获得作品版权

优先级：最高
状态：基本完成，后续继续零星检查

---

### 2. 登录 / 注册页面改成小窗样式

当前问题：

* 登录注册页现在是完整大页面。
* 对标站点更像居中小窗，没有太重的页面感。

处理方向：

* 保留 `/login` 和 `/register` 路由
* 视觉上改成居中小窗
* 背景可以保留暗色剧场风格
* 表单区域更紧凑
* 后期可再做弹窗登录，但当前不强求

优先级：高
状态：部分完成，已接入登录 / 注册拦截弹窗，独立页面后续再统一视觉

---

### 3. 前台页面统一 Navbar

当前问题：

* 博客 / 内容站通常所有前台页面都应该有顶部导航栏。
* 目前部分页面可能已有，后续需要统一检查。

处理方向：

* 前台页面统一保留 Navbar
* 后台页面使用独立后台管理导航，不一定使用前台 Navbar
* 登录页 / 注册页可根据设计决定是否显示完整 Navbar，至少要有返回首页入口

优先级：高
状态：已完成，前台主要页面已统一使用 Navbar

---

### 4. 清理开发痕迹

当前问题：

* 页面中可能还存在“稍后开发”“下一步开发”“内部 ID”等临时文案。
* 左下角可能出现 Next.js 开发环境的 “N” 图标。
* 某些禁用按钮属于开发占位，不应长期存在。

处理方向：

* 全局搜索并清理：

  * 稍后开发
  * 下一步开发
  * 内部 ID
  * 临时按钮
* “模拟支付”暂时保留，因为真实支付尚未接入
* 左下角 N 图标确认是否为 Next.js 开发环境标记；生产环境通常不会出现

优先级：高
状态：部分已做，继续检查

---

### 5. Toast / Confirm 组件替代浏览器弹窗

当前问题：

* 当前大量使用 `alert()` 和 `confirm()`
* 浏览器原生弹窗不美观，也不符合网站整体风格

处理方向：

* 创建 Toast 提示系统
* 创建自定义 Confirm 确认弹窗
* 逐步替换：

  * 登录成功 / 失败
  * 注册成功 / 失败
  * 发布成功
  * 编辑成功
  * 删除确认
  * 取消订单确认
  * 支付成功 / 失败

优先级：高
状态：部分完成，核心互动已使用 FeedbackProvider，旧登录/注册/部分后台操作仍需后续替换

## 近期开发路线

### 6. 后台 UI 美化

当前问题：

* 前台页面已经开始统一风格
* 后台页面仍有部分旧样式

处理顺序：

1. `/dashboard` 后台首页
2. `/dashboard/posts` 内容管理页
3. `/dashboard/upload` 发布页面
4. `/dashboard/posts/[id]/edit` 编辑页面
5. `/dashboard/orders` 订单管理页
6. `/dashboard/purchases` 购买权限页

设计方向：

* 保持暗色风格
* 更偏管理工具感
* 卡片、按钮、状态标签统一
* 管理入口清晰
* 表单区域更舒服
* 订单 / 帖子列表更易读

优先级：高
状态：进行中

### 11. 评论回复、点赞、消息提醒

需求：

* 普通用户评论可以被回复
* 评论可以点赞
* 用户收到回复提醒
* 系统通知和互动通知分开

拆分为：

#### 11.1 评论回复

* 新增 `parentId`
* 支持一级回复或多级回复
* 评论区按层级显示

#### 11.2 评论点赞

* 新增 `CommentLike`
* 一个用户对同一评论只能点赞一次

#### 11.3 消息提醒

* 新增 `Notification`
* 类型：

  * 评论回复
  * 评论点赞
  * 系统通知
  * 作品更新
* Navbar 或个人中心显示未读数量

优先级：中
状态：部分完成，评论回复、回复通知、通知中心和 Navbar 未读数已完成；评论点赞和系统通知后续再做

---

### 12. 作品点赞与收藏夹

需求：

* 作品帖子可以点赞
* 作品帖子可以收藏
* 用户有收藏夹
* 未购买的付费作品即使收藏，也不能查看隐藏内容

处理方向：

* 新增 `PostLike`
* 新增 `Favorite`
* 个人中心增加“我的收藏”
* 收藏只代表保存入口，不代表购买权限
* 查看付费内容仍然以 `Purchase` 为准

优先级：中
状态：已完成，已支持作品点赞、作品收藏、我的收藏和卡片互动入口

## 邮箱系统与账号安全

### 14. 邮箱验证码注册

需求：

* 注册时向邮箱发送验证码
* 用户填写验证码
* 校验通过后才能注册

处理方向：

* 接入 SMTP 或邮件服务
* 新增 `EmailVerificationCode` 表
* 验证码有效期，例如 10 分钟
* 同邮箱限制发送频率
* 注册 API 校验验证码

优先级：上线前高
状态：待做

---

### 15. 找回密码

需求：

* 用户输入邮箱
* 系统发送验证码或重置链接
* 验证通过后设置新密码

处理方向：

* 可复用邮箱验证码机制
* 新增 `/forgot-password`
* 新增 `/reset-password`
* 重置后清除旧 session

优先级：上线前高
状态：待做

---

### 16. 邮件订阅与作品更新通知

需求：

* 注册时用户可选择是否接收邮件通知
* 个人中心可修改是否接收邮件通知
* 发布作品或更新作品时，管理员可选择是否通知购买过任何付费作品的用户

处理方向：

* `User` 表增加 `receiveEmailUpdates Boolean @default(false)`
* 发布 / 编辑作品时增加“邮件通知用户”选项
* 收件人范围：

  * 曾经有成功付费订单的用户
  * 且开启邮件通知
* 邮件发送应排队或分批，避免一次性阻塞请求

优先级：中后期
状态：待做

## 支付系统

### 17. 樱奈 EPAY 接入预研

已知方向：

* 使用商户订单号 `orderNo`
* 将来可对应支付接口的 `out_trade_no`
* 平台订单号保存到 `providerTradeNo`
* 支付方式保存到 `paymentType`
* 支付成功后必须以后端异步通知为准
* 验签成功后才能把 Order 改为 PAID 并创建 Purchase

当前状态：

* Order 表已存在
* Purchase 表已存在
* orderNo 已加入
* 真实支付暂未接入
* 当前仍为模拟支付

优先级：上线前高
状态：预留中
