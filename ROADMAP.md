# AmekoRyTheater Roadmap

本文件用于记录 AmekoRyTheater 的 MVP 发布前状态、路线取舍和后续开发顺序。

项目定位：个人创作者内容站，核心模式是免费作品展示 + 单篇付费作品购买解锁。当前不做包月会员制，不做当月作品 / 过往作品区分，也不在用户界面使用“买断”来描述购买行为。

---

## MVP 当前状态

### 已完成主干

* Next.js App Router + React + TypeScript + Tailwind CSS。
* PostgreSQL + Prisma 数据层。
* Session 登录态、普通用户 / 管理员角色、后台权限保护。
* 登录、退出、注册邮箱验证码。
* 找回密码与重置密码，重置成功后清理旧 session。
* 首页、作品列表、作品详情、公告、关于页。
* 作品帖 / 公告帖，支持免费 / 付费、草稿 / 发布、置顶。
* 封面图和作品图集上传、编辑、删除、详情页轮播。
* 评论、一级回复、评论删除权限。
* 作品点赞、收藏、个人收藏页。
* 订单、待支付、取消、模拟支付、购买权限、付费内容解锁。
* 个人中心、头像上传、订单记录、购买记录、我的评论。
* 站内通知中心、未读数、单条 / 全部已读。
* 管理员收到评论、点赞、收藏、购买相关站内通知。
* 邮件系统统一出口：dev-console 与 Resend。
* 邮件验证码防刷、REGISTER / RESET_PASSWORD purpose 隔离。
* 账户设置页，可保存邮件通知偏好。
* 评论回复邮件通知和购买成功邮件通知。
* 后台总览、内容管理、发布、编辑、订单、购买权限、站点设置。
* SiteSetting：站点标题、副标题、滚动公告、关于内容、联系信息、Footer 外链、首页背景图、Hero 图。
* FeedbackProvider toast / confirm 已覆盖核心交互。

### 当前仍是开发态的边界

* 支付仍是模拟支付，不可用于真实收费上线。
* 上传文件仍在本地 `public/uploads`，生产部署需要持久化存储策略。
* `dev-console` 邮件模式只适合本地测试；真实发信需要 Resend 域名、额度、退信和垃圾箱表现确认。
* `PasswordResetToken` 模型是预留，当前找回密码使用邮箱验证码。
* 新作品发布邮件群发只保留偏好字段，尚未接入发送逻辑。
* 评论点赞、系统公告类通知、退款流程、真实支付回调审计尚未做。

## 三类用户入口检查

### 游客

主要入口完整：

* Navbar：首页、作品、公告、关于、登录、注册。
* 首页：展示作品和站点视觉内容。
* 作品列表 / 详情：可浏览免费内容和付费作品简介。
* 登录 / 注册：支持独立页面和站内拦截弹窗。
* 忘记密码入口：登录页和登录弹窗可进入。

需注意：

* 游客不能评论、点赞、收藏、购买，应继续保持友好提示和登录引导。
* 付费内容必须继续只展示预览，不暴露隐藏内容和下载信息。

### 普通用户

主要入口完整：

* 头像菜单：个人中心、我的通知、账户设置、退出登录。
* 个人中心：头像、账号信息、订单、购买记录、评论、收藏、通知、账户设置入口。
* 作品详情：评论 / 回复、点赞 / 收藏、购买、订单跳转、付费内容解锁。
* 订单详情：待支付订单可取消 / 模拟支付，已支付订单可回到作品。
* 账户设置：邮件通知偏好保存。

需注意：

* 普通用户不能访问后台。
* 普通用户只能管理自己的订单、评论、通知和账户设置。

### 管理员

主要入口完整：

* Navbar 登录后显示“后台”。
* 后台首页：总览和管理入口。
* 后台内容管理：发布、编辑、删除、查看作品 / 公告。
* 后台订单管理：查看订单状态、订单号、支付信息。
* 后台购买权限：查看已解锁记录。
* 后台站点设置：站点文案、滚动公告、视觉资源、联系方式和外部链接。

需注意：

* 管理员可以排查订单，但不应替普通用户模拟支付。
* 后台可显示内部 ID；前台用户页尽量不显示内部 ID。

## 关键链路状态

### 注册验证码

状态：已完成。

* 注册前必须发送并填写 `REGISTER` 验证码。
* 验证码正确、未过期、未使用才允许注册。
* 同邮箱同 purpose 有 60 秒冷却和短窗口防刷。
* 已注册邮箱发送注册验证码返回友好提示。

### 登录 / 退出

状态：已完成。

* Session cookie 登录。
* 管理员登录后进入后台。
* 普通用户登录后回前台。
* 退出登录会删除 session 并清 cookie。

### 找回密码

状态：已完成。

* 发送 `RESET_PASSWORD` 验证码。
* 不暴露邮箱是否存在。
* 重置密码需要邮箱、验证码、新密码、确认密码。
* 重置成功后清理旧 session。

### 作品浏览

状态：已完成。

* 首页、作品列表、公告列表、详情页可访问。
* 免费 / 付费内容展示边界已建立。
* 已购买用户可查看付费内容和下载信息。

### 评论 / 回复

状态：已完成第一轮。

* 登录用户可评论和一级回复。
* 回复自己不生成通知 / 邮件。
* 删除权限区分管理员和本人。
* 多级深层嵌套暂不做。

### 点赞 / 收藏

状态：已完成。

* 作品点赞和收藏支持幂等切换。
* 收藏页已接入个人中心。
* NOTICE 不参与作品互动。

### 购买 / 解锁

状态：开发闭环完成，但真实支付未完成。

* 订单创建、取消、模拟支付、购买权限、付费内容解锁已闭环。
* `finalizePaidOrder` 作为支付成功统一收口。
* 真实收费上线前必须接入支付 provider 回调验签。

### 站内通知

状态：已完成第一轮。

* 评论回复通知。
* 管理员收到评论、点赞、收藏、购买相关通知。
* 通知中心支持单条 / 全部已读。
* Navbar 显示未读数。

### 邮件通知

状态：已完成第一轮。

* 注册验证码、找回密码验证码。
* 评论回复邮件通知。
* 购买成功 / 内容已解锁邮件通知。
* 账户设置可关闭对应偏好。
* 新作品发布群发暂不接入。

### 账户设置

状态：已完成第一轮。

* `/profile/settings` 可保存三类邮件通知偏好。
* 未登录访问跳转登录，API 返回 401。

### 后台内容管理

状态：MVP 可用。

* 发布 / 编辑 / 删除作品与公告。
* 图集上传、选择、删除。
* 订单、购买权限、站点设置页已接入。
* 后台 UI 可继续打磨，但不是阻塞业务闭环的问题。

## 明显开发痕迹和过时说明

### 上线前应清理

* `app/api/artworks/route.ts` 看起来是早期测试接口：只打印请求体并返回“作品创建成功”，当前未发现被正式表单引用。建议上线前删除或改成正式用途。
* 前台支付文案仍是“模拟支付”。如果要真实收费上线，必须先接真实支付；如果只是内部预览上线，可以明确标记为测试环境。
* `PAYMENT_FLOW.md` 明确说明当前为 simulated payment only，这一点必须保留到真实支付接入完成。
* 邮件真实发送前，需要确认 `.env.local` / 部署环境里的 `SITE_URL`、`EMAIL_PROVIDER`、`RESEND_API_KEY`、`EMAIL_FROM`。
* `EMAIL_FLOW.md` / `CODEX_CONTEXT.md` 旧段落中仍保留大量历史 update record，后续可以归档，但当前不影响运行。

### 可以暂时保留

* 后台订单页展示“内部 ID”属于排查信息，可保留。
* `dev-console` 邮件 provider 是本地开发工具，应保留。
* `PasswordResetToken` 模型目前是预留，不必急删。
* `console.error` 用于服务端错误日志，可以保留。

## MVP 上线前必须做

1. 真实支付决策

   * 如果要真实收费上线：必须接入 EPAY 或其他支付 provider。
   * 必须以后端 notify / callback 验签为准完成支付。
   * 必须确认金额、订单号、支付状态、重复回调幂等。
   * return_url 只能展示结果，不能决定支付成功。

2. 生产环境配置

   * 生产 PostgreSQL。
   * HTTPS 和正式域名。
   * `SITE_URL` 设置成正式域名。
   * Resend 域名验证、发信额度、退信和垃圾箱表现确认。
   * 环境变量只放服务器，不提交真实 key。

3. 上传文件持久化

   * 当前图片在本地 `public/uploads`。
   * 部署到非持久文件系统时需要对象存储、挂载盘或备份策略。

4. 开发痕迹清理

   * 处理 `app/api/artworks/route.ts` 早期测试接口。
   * 全站再扫一遍“稍后开发 / 下一步开发 / 买断 / alert / window.confirm”。
   * 确认前台不展示内部 ID。

5. 冒烟测试清单

   * 游客浏览免费 / 付费作品。
   * 注册验证码、登录、退出、找回密码。
   * 评论、回复、点赞、收藏。
   * 下单、取消、支付、解锁。
   * 站内通知、邮件通知偏好、邮件发送。
   * 管理员发布、编辑、删除、站点设置。

---

## 可以上线后再做

* 后台 UI 继续美化和表格筛选。
* 订单搜索、按订单号 / 用户邮箱筛选。
* 评论点赞。
* 系统通知和作品更新站内通知。
* 新作品发布邮件通知，但需要分批 / 队列，不建议在发布请求里同步群发。
* favicon 后台上传；近期可以先手动替换图标。
* 更完整的用户资料页。
* 图片存储迁移到对象存储后的管理界面。
* 支付退款、退款通知、退款权限回收策略。
* 更细的管理员权限分级。

---

## 暂时不建议做

* 包月会员制、年度合集、当月作品 / 过往作品区分。
* 复杂推荐算法。
* 复杂多级评论树。
* 实时 WebSocket 通知。
* 多管理员团队协作。
* 大规模 UI 重构或设计系统重写。
* 新作品邮件群发的同步实现。

---

## 下一阶段建议路线

### Phase 1：MVP 发布前收口

1. 删除或处理 `app/api/artworks/route.ts` 测试接口。
2. 再跑一次全站文案和开发痕迹扫描。
3. 明确上线类型：内部预览 / 真实收费。
4. 如果真实收费，优先接入真实支付回调验签。
5. 准备生产环境变量和文件存储方案。
6. 做完整手动冒烟测试。

### Phase 2：真实支付与部署

1. 接入 EPAY 下单 / notify_url / 验签。
2. 记录平台流水号、支付方式和回调审计信息。
3. 部署生产数据库、HTTPS、域名和备份。
4. 确认 Resend 真实发信表现。

### Phase 3：上线后体验增强

1. 后台订单搜索和筛选。
2. 后台 UI 继续统一。
3. 评论点赞和系统通知。
4. 新作品发布通知，使用分批或队列策略。
5. favicon 和品牌细节。
